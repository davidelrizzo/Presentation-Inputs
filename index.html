Number

Exactly what is a valid number?
1 | -100.23 | 00001.00 | 1234a | 1,234.00[1] | 01 02 03[2] | -6.23e01

iOS 5+6 allows a comma (exactly) on the thousands mark but does not submit it

Spaces are valid in Chrome & IE, surrounding spaces only in Safari & iOS
IE is the only one that submits them

Number

The min, max and step value

An undefined step value defaults to 1.
With step="any" any floating point number is valid

Number

Touch keypads

Using the (invalid) pattern attribute "[0-9]*" invokes the iOS Integer keypad

DO NOT use iOS pattern if negative or decimal values are possible!

Numeric codes

So what about Postcodes, Pins, Credit Cards &
Serial numbers, etc..?


Anything where each character is more important than value submitted

There is no HTML5 'type' for these!

If input uses ONLY DIGITS type="tel" is the only way to invoke the numeric keypad on all devices

Allow desktop users to enter the value however they like

Dates

So many ways of inputting dates:

[text field] [selects] [tabbed input] [masked input] [datepicker] [native] [device native]

Native date input always submit in YYYY-MM-DD format

Best for Date of Birth: tabbed/masked
Best for 'near' date: datepicker
